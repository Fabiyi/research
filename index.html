HISTORY OF WEB

It was an idea introduced by Paul Baran and Donald Davies independently developed concepts for
packet-switched networks, which would later be essential to the internet. ARPANET funded by the
U.S. Department of Defense, because then first operational packet-switching network in 1969. In 1974,The Internet was introduced by Vint Cert and Bob Kahn who developed the Transmission Control Protocol (TCP)
and Internet Protocol(IP), which are the fundamental protocols of the internet. It is also called world wide web, is a fascinating journey that revolutionized communication, information sharing,
and global connectivity. It was invented by Tim Berners-Lee, a British scientist in 1989. He proposed the idea of a "hypertext" system that would link document together. He developed key components of the web.

  1. HTML (Hypertext Mark-up Language): It is the language used to create web page.
  2. HTTP (Hypertext Transfer Protocol): It is the protocol used to transfer data over the web.
  3. URL (Uniform Resource Locator): It is the address system used to locate web page.

The first website went live on August 6, 1991 which explain the concept of the World Wide Web. The release of web browsers such as Mosaic web browser, Netscape Navigation, Internet Explorer, JavaScript, Google, Social Web such as wikis Facebook, YouTube, X(twitter),Mobile & Cloud era such as Amazon , Google Cloud ,HTML,CSS3 & JavaScript frameworks,Web3.0 such as AI , machine learning and block chain . It brought about growth of commercialization with the modern web era.

VERSION CONTROL

It is a system or practice that manages changes to documents, computer programs, large websites, or other collections of information.
It keeps track of every modification made to the content, allowing you to revert to previous version, compare changes over time,
and collaborate with others without conflicting updates.
A database that stores files and their revision history in a local(on the computer) or remote(on a server)is called REPOSITORY.
A record of changes made to files in the repository is called COMMIT.
A separate line of development that diverges from the main project is called BRANCH.
A situation where different changes have been made to the same part of a file in different branches is called CONFLICT.

TYPES OF VERSION CONTROL SYSTEM

Central Version Control System (CVCS)

It stores all files and their revision history in a central server. The users check out files and work on them, then commit
changes back on the server. Pros: Easier to understand, single source of truth. E.g. Subversion (SVN). Cons: If the server goes down, no one can collaborate, and you need a network connection to commit or update.

Distributed/Decentralized Version Control System (DVCS)

It enable the user make changes locally and later push them to a remote server.
Every user has a full copy of the repository

Pros: Allow offline work, supports multiple workflows (like feature branches), and is more resilient to server.

Cons: Slightly more complex to learn and manage due to distributed nature.
